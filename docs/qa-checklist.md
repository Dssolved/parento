# QA checklist для Parento

Этот чеклист нужен после наполнения Supabase демо-контентом из `supabase/demo-content.sql`.

## Подготовка

- Запустить локально: `npm run dev`.
- Открыть `http://localhost:5173/`.
- Убедиться, что Supabase env-переменные заполнены.
- Выполнить `supabase/demo-content.sql` в Supabase SQL Editor.
- Для чистого теста создать нового пользователя или выйти из текущего аккаунта.

## Новый Free-пользователь

- Зарегистрироваться новым email.
- Подтвердить email, если подтверждение включено.
- Войти в аккаунт.
- Пройти onboarding:
  - выбрать этап;
  - выбрать роль;
  - увидеть финальный экран со стартовым курсом.
- Нажать `Перейти к первому уроку`.
- Открыть урок и отметить его как пройденный.
- Нажать `Следующий урок`.
- Вернуться в `Моё обучение`.
- Проверить:
  - число пройденных уроков увеличилось;
  - блок продолжения ведёт к следующему уроку;
  - профиль показывает выбранный этап и роль.
- В отдельном тестовом аккаунте проверить удаление аккаунта из профиля:
  - кнопка активируется только после ввода `УДАЛИТЬ`;
  - после удаления происходит выход на главную;
  - пользователь исчезает из Authentication;
  - профиль и прогресс удалены из таблиц.

## Free-пользователь и Premium-доступ

- Открыть каталог.
- Найти Premium-курс.
- Открыть страницу Premium-курса.
- Проверить:
  - виден баннер `Premium доступ`;
  - кнопка ведёт на страницу подписки;
  - уроки курса заблокированы.
- Открыть Free-курс со смешанным доступом:
  - `Старт перед беременностью`;
  - `Беременность спокойно по триместрам`;
  - `Первый месяц с малышом`.
- Проверить, что первые уроки доступны, а третий урок помечен Premium и ведёт на подписку.

## Premium-пользователь

- Перейти на страницу `Подписка`.
- Нажать `Активировать Premium`.
- Проверить, что статус стал `Premium`.
- Открыть Premium-курс.
- Проверить:
  - баннер блокировки исчез;
  - уроки открываются;
  - уроки можно отмечать пройденными;
  - дашборд предлагает продолжить обучение.

## Прогресс

- Пройти 2-3 урока в одном курсе.
- Открыть страницу курса.
- Проверить:
  - процент курса изменился;
  - отображается `N из M уроков`;
  - пройденные уроки имеют бейдж `Пройдено`.
- Пройти последний урок курса.
- Открыть дашборд.
- Проверить состояние завершённого курса.

## Админка

- Войти под admin-профилем.
- Открыть `/admin`.
- Создать новый курс.
- Добавить 2 урока.
- Проверить публикацию:
  - новый курс и новый урок по умолчанию создаются как черновики;
  - черновик курса не виден в каталоге;
  - черновик урока не виден на странице опубликованного курса;
  - опубликованный урок внутри черновика курса не открывается по прямой ссылке.
- Проверить Markdown:
  - заголовок;
  - список;
  - таблицу;
  - режим предпросмотра.
- Добавить URL обложки.
- Отметить урок как Premium.
- Проверить в пользовательском режиме, что доступ к уроку закрыт для Free.
- Отредактировать урок.
- Удалить тестовый урок.

## Визуальная проверка

- Проверить страницы на desktop:
  - главная;
  - onboarding;
  - дашборд;
  - каталог;
  - курс;
  - урок;
  - подписка;
  - профиль;
  - админка.
- Проверить основные страницы на мобильной ширине.
- Особое внимание:
  - кнопки не переносят текст некрасиво;
  - карточки курсов не прыгают из-за картинок;
  - footer не перекрывает контент;
  - длинные названия уроков не ломают сетку.

## Мобильная версия

- Проверить, что мобильный viewport действительно применяет мобильную верстку после:
  - обычного открытия страницы;
  - refresh;
  - перехода между страницами;
  - открытия DevTools/device mode;
  - возврата с desktop-ширины на mobile-ширину.
- Зафиксировать страницы, где вместо мобильной верстки остаётся desktop-раскладка.
- Проверить хэдер на мобильной ширине:
  - текущая навигация выглядит перегруженной;
  - нужен burger-menu для ссылок `Моё обучение`, `Курсы`, `Подписка`, `Профиль`, `Админ`;
  - в открытом меню должны быть понятные состояния активной страницы и кнопка выхода.
- Проверить, что хэдер не переносит элементы некрасиво и не занимает слишком много высоты.

## После проверки

- Выписать найденные UX-проблемы отдельным списком.
- Разделить их на:
  - критично для пути пользователя;
  - визуально неприятно;
  - можно отложить.
